# 记事模块进度文档
> 当前模块的产品依据来自 `docs/app/design.md` 中的 `4.1 记录类对象`、`5.3 记事页`、`8.3 记事页线框`、`9.2 记事`。

## 当前状态

- 已补齐记事模块独立进度入口，后续进度统一在本文件维护。
- 当前实现已经从前端只读 mock 时间线切到后端 `events.json` 种子与保存命令。
- 页面已支持：
  - 事件快速录入
  - 时间线浏览
  - 单条事件的简要说明
  - 管理模式下编辑 / 删除
  - 国际化文案
  - 主题变量适配
  - 固定布局与堆叠布局两套响应式展示

## 已完成

- 新增 Tauri 侧 `events` 模块，使用 `events.json` 做最小持久化，并提供 `get_events` / `save_events` 命令。
- 将记事样例数据收敛到 Rust `seed.json`，删除前端事件 mock 文件。
- 记事页改为真正的一页式工作台：右侧快速录入 + 主题概览，左侧时间线。
- 新增事件编辑弹窗，保存时写回完整后端模块数据，避免搜索态编辑误删隐藏记录。
- 补齐 `zh` / `en` 的 `events.*` 文案键。
- 为三套主题补齐记事模块强调色变量。
- 扩展 `EventEntry` 结构，补入 `createdAt` / `occurredAt` / `content` / `tags` / `status`，与 app 设计文档的记录对象约定对齐。

## 待处理

- 如后续需要在纯浏览器 mock 模式下继续演示记事模块，可再决定是否让 mock API 复用同一份后端 seed。
- 如后续需要跨模块联动，可将记事与反思 / 关系 / 成长记忆做引用关联。
