# BetterToLive V1 页面结构与线框说明

## 1. 文档目的

本文档承接 [product-requirements.md](/Users/apple/Documents/github/bettertolive/docs/product-requirements.md)，用于继续明确 BetterToLive V1 的页面结构、页面职责、核心流转和低保真线框。

这份文档重点解决四件事：

1. V1 到底有哪些页面
2. 每个页面需要承接什么内容
3. 菜单应该如何分组，避免变成零散工具箱
4. 首页和各模块应该怎样组织成一个统一体验

## 2. V1 页面清单

当前版本先收敛为 `1 个首页 + 9 个核心模块页 + 1 个通用快速记录入口`。

### 2.1 页面列表

1. `首页 / 总览`
2. `反思页`
3. `记事页`
4. `记账页`
5. `购物清单页`
6. `观念页`
7. `原则页`
8. `关系页`
9. `成长页`
10. `未来蓝图页`
11. `快速记录弹层`

### 2.2 导航方式

- 左侧主导航常驻，但不再只是平铺入口，而是分成 `工作台 / 记录 / 自我图谱 / 方向` 四组
- 首页是总览页，不是唯一工作流入口
- 各模块之间可横向切换，不要求用户回首页再继续
- 快速记录入口在所有页面都可触达，用于降低输入门槛

## 4. 共享对象建议

V1 虽然不细化数据库设计，但页面结构需要依赖一组统一对象观念。

### 4.1 记录类对象

所有记录类内容建议默认包含以下字段：

- `id`
- `type`
- `createdAt`
- `occurredAt`
- `content`
- `tags`
- `status`

### 4.2 模块对象补充

- `反思记录`：可带 `mood`、`summary`
- `事件记录`：可带 `title`
- `账目记录`：可带 `amount`、`direction`、`category`
- `购物项`：可带 `stage`、`priceExpectation`
- `未来蓝图项`：可带 `theme`、`targetState`、`nextStep`

## 5. 关键页面职责

### 5.1 首页 / 总览

**目标**

- 帮用户在最短时间进入今天最想做的事
- 同时让用户感到自己正在看的是一个完整人生系统，而不是工具集合

**必须承接的内容**

- 顶部欢迎语与当天时间
- 一组可直接点击的主入口
- `形成 / 过去 / 现在 / 未来` 摘要
- 最近记录列表
- 页面逻辑说明
- 快速记录按钮

**不承担的内容**

- 不做复杂编辑
- 不替代模块页完成深度输入

### 5.2 反思页

**目标**

- 成为日常最自然的表达入口

**必须承接的内容**

- 新建反思输入区
- 最近反思列表
- 按日期回看

### 5.3 记事页

**目标**

- 让事件和经历形成连续时间线

**必须承接的内容**

- 事件快速录入
- 时间线浏览
- 单条事件的简要说明

### 5.4 记账页

**目标**

- 让支出和收入成为理解现实生活的证据

**必须承接的内容**

- 快速记一笔
- 本周 / 本月摘要
- 账目列表

### 5.5 购物清单页

**目标**

- 让用户看到自己的欲望、需要和购买决策

**必须承接的内容**

- 新建购物项
- `想买 / 待买 / 已买` 三列或三段视图
- 状态切换

### 5.6 未来蓝图页

**目标**

- 给用户一个长期定义自我和生活方向的地方

**必须承接的内容**

- 理想自我
- 理想生活
- 重要价值
- 阶段路径

### 5.7 观念页

**目标**

- 明确承接 `人生观 / 世界观 / 价值观`
- 让用户以后可以理解很多判断背后的来源

**必须承接的内容**

- 世界观
- 人生观
- 价值观
- 反复出现的问题

### 5.8 原则页

**目标**

- 把原则、底线、边界从模糊感受里拿出来

**必须承接的内容**

- 原则清单
- 不愿再退让的底线
- 原则形成来源

### 5.9 关系页

**目标**

- 让关系不再只是“联系人”，而是影响自我认知的重要结构

**必须承接的内容**

- 家庭与亲戚关系
- 朋友与同伴关系
- 重复出现的关系模式

### 5.10 成长页

**目标**

- 通过成长环境、接触过的人和事，理解自己为什么会成为现在这样

**必须承接的内容**

- 按阶段的成长线索
- 每个阶段的环境与影响
- 至今仍在起作用的痕迹

## 6. 全局交互原则

- 用户进入任何页面后，都应能在 1 次点击内切换到其他模块
- 所有页面都应提供一个明显的 `快速记录` 入口
- 列表类页面优先展示最近内容，再考虑过滤或分类
- 菜单副标题默认单行展示，不让单条菜单高度继续膨胀
- V1 默认先服务“写下来”“能回看”“能解释自己”，而不是高级分析

## 7. 核心流转

### 7.1 日常轻量使用流

```text
打开应用
→ 看首页摘要
→ 选择直接写反思 / 记一笔 / 记一件事 / 加购物项 / 看未来蓝图
→ 完成一次输入
→ 返回原页面继续浏览或切换到其他模块
```

### 7.2 回看自我流

```text
打开应用
→ 从首页进入最近记录
→ 查看反思或事件
→ 再跳到记账或未来蓝图对照
→ 形成“过去 - 现在 - 未来”的自我理解
```

### 7.3 从现实回到理想流

```text
先看近期支出或购物项
→ 感知自己最近在把资源投向哪里
→ 打开未来蓝图页
→ 对照理想生活与当下生活是否一致
```

## 9. 页面优先级建议

如果后续开始实现，推荐优先级如下：

1. 首页
2. 反思页
3. 未来蓝图页
4. 记事页
5. 记账页
6. 购物清单页
7. 快速记录弹层

这样做的原因是：首页负责统一心智，反思页负责最常用输入，未来蓝图页负责把产品从普通记录工具拉回到产品主线。
