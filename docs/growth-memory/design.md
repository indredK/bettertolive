# 成长记忆模块设计文档
> 汇总 成长记忆模块 的模块定位、分类维度、页面/数据设计和边界。

---

## 成长记忆模块设计

### 成长记忆模块 - 人生节点与成长轨迹分类建议

#### 1. 30 秒看懂这份文档

这份文档建议给每一条成长节点 / 人生记忆打 **5 个分类标签** + **1 个评估属性**：

##### 5 个分类标签（用来归类、过滤、看人生轨迹）

| 标签 | 它回答什么 | 取值示例 |
|---|---|---|
| **type**（内容类型） | 这是一段什么类型的记忆？ | 事件 / 地点 / 物件 / 人物 / 照片 / 领悟（共 6 个，见 §4） |
| **era**（人生时期） | 它属于哪段人生？ | 童年 / 学生时代 / 初入社会 / 某段关系期 / 某座城市期 / 当下 |
| **emotional_weight**（情感重量） | 它现在碰起来重不重？ | 轻 / 中性 / 重 / 很重 |
| **processing**（整理状态） | 我已经理解它了吗？ | 已整理 / 正在理解 / 暂不触碰 / 决定不再细究 / 开放问题 |
| **privacy**（隐私级别） | 谁能看到？ | 仅自己 / 指定的人 / 未来可公开 / 离世后可看 |

##### 1 个评估属性（跟着每条记忆走，用来衡量"这一条"）

| 属性 | 它回答什么 | 取值示例 |
|---|---|---|
| **formative_power**（塑造力） | 这段经历对我今天的塑造有多深？ | 极深 / 较深 / 中等 / 轻微 / 无 |

> **5 维和塑造力的区别**：5 维用来**归类**（"学生时代我都经历了什么"）；塑造力用来**评估**（"这段经历到底改变了我多少"）。详见 §7。

打满这 5 个标签 + 1 个属性后，原来问不出的问题就能问出来了：

- "我的人生转折点都在哪里？"
- "哪段时期对我的塑造最深？"
- "还有哪些记忆没整理好，该去面对了？"
- "我是怎么一步步变成现在的自己的？"

#### 2. 为什么不只按时间线排列

如果只按时间顺序排列记忆，会出现这样的列表：

> 2003 搬家 / 2008 升学 / 2012 分手 / 2015 换城市 / …

这种分类的问题：

1. **看不见深层结构**——2003 搬家 和 2015 换城市 其实都是"地点锚点"，时间线把它们分开了
2. **感受不到轻重**——一次分手可能比十次搬家更重，但时间线上它们都是"一个点"
3. **漏掉了非事件记忆**——老家的气味、妈妈的一道菜、一张照片里的光，这些不是"事件"，但比事件更难分类
4. **无法追踪整理进度**——知道有 200 条记忆，不知道哪些已经理解了、哪些还不敢碰

##### 换一个问法

不要问"什么时候发生的"，问"**它是哪种类型的记忆？它现在碰起来重吗？我已经理解它了吗？**"

> **什么叫"整理状态"**——不是所有过去都需要被理解。有些记忆已经安放好了，有些还很沉重不适合频繁打开，有些可能永远只能承认存在而无法真正理解。给每种状态一个合法位置，比默认"所有记忆都应该被整理完"更诚实。

##### 按这套维度分类的好处

- **按类型找记忆**：想知道"我生命里重要的地点"时不需要遍历时间线
- **知道哪些还没处理**：processing 维度让未整理的记忆不被遗忘但也不被催促
- **看塑造关系**：formative_power 高的记忆集合，就是"是什么造就了我"的答案
- **隐私分层**：敏感记忆可以标记 privacy 级别，不同级别不同处理

#### 3. 用两条具体记忆走一遍

##### 例 1：第一次离开家去另一个城市

**5 个分类标签**：

| 维度 | 取值 | 为什么 |
|---|---|---|
| type | **事件** | 它是一个具体发生过的经历 |
| era | **初入社会** | 大学毕业后的第一个转折 |
| emotional_weight | **中性** | 当时很重，现在已经释然了 |
| processing | **已整理** | 已经回看过几次，理解它意味着什么 |
| privacy | **仅自己** | 有一些脆弱细节，不打算给别人看 |

**评估属性**：

| 属性 | 取值 | 含义 |
|---|---|---|
| formative_power | **极深** | 那一次离开定义了自己后来"怎么面对陌生环境"的模式 |

> 现在你可以问：我**初入社会**（era）时期有哪些**formative_power=极深**（塑造力）的记忆？这些是理解"我为什么是现在这样"的关键。

##### 例 2：奶奶家的厨房

**5 个分类标签**：

| 维度 | 取值 | 为什么 |
|---|---|---|
| type | **地点** | 它不是事件，而是一个反复回去的空间 |
| era | **童年** | 最早的记忆锚点之一 |
| emotional_weight | **重** | 奶奶已经不在了，这个地方碰起来还是会难过 |
| processing | **已整理** | 曾经写过长文，基本安放好了 |
| privacy | **指定的人** | 希望家人能看到，但不想公开 |

**评估属性**：

| 属性 | 取值 | 含义 |
|---|---|---|
| formative_power | **较深** | 那个厨房定义了自己对"家"和"温暖"的最早感受 |

> 两条记忆类型不同、时期不同、塑造力不同——但都是"已整理"且 formative_power 很高。这组记忆就是你人生中"已经消化了的重大影响源"。

#### 4. 第 1 维：6 种记忆类型

##### 为什么是 6 种

不是按"重要/不重要"分，而是按**记忆的承载形式**来分。不同类型的记忆需要不同的记录方式和唤起路径。

| 类型 | 承载形式 | 典型条目 | 记录时重点 |
|---|---|---|---|
| **事件** | 一件具体发生的事 | 搬家、升学、分手、重大决定、第一次 | 发生了什么、当时感受、后来影响 |
| **地点** | 一个反复出现的空间 | 老家、学校、出租屋、医院、车站、街道 | 那个空间的样子、气味、光线、当时谁在 |
| **物件** | 一件有情感附着的物品 | 旧玩具、一封信、一件衣服、一件礼物 | 物品的来历、为什么留着、放下会怎样 |
| **人物** | 一个曾经重要的人 | 已故亲人、旧友、老师、短暂相遇的人 | 这个人对自己的影响、还没说的话 |
| **照片** | 一张有故事的照片 | 老照片、截图、某次旅行的合影 | 照片里的场景、当时的心境、现在回看的感受 |
| **领悟** | 一次认知上的转变 | "那天我突然明白……""那次之后我再也不……" | 转变前相信什么、转变后相信什么、什么触发了转变 |

##### 类型的交叉

一条记忆可能同时涉及多个类型。比如"和奶奶在厨房的合影"——既是照片也是地点也是人物。归属规则：**选记录时最主要想通过什么承载形式来讲述这段记忆**。

- 主要想讲那个厨房的样子 → 地点
- 主要想讲和奶奶的关系 → 人物
- 主要想讲那张照片本身 → 照片

#### 5. 第 2 维：era（人生时期）

`type` 回答"以什么形式承载"，`era` 回答"属于哪段人生"。

##### 时期的定义

时期不是固定枚举，而是用户自己定义的人生分段。因为它高度个人化——有人用"城市"分段（北京时期、上海时期），有人用"关系"分段（在一起的那几年），有人用"阶段"分段（读书、工作、自由职业）。

建议给几个默认分段提示，但允许自定义：

| 默认分段提示 | 覆盖什么 |
|---|---|
| 童年（0-12 岁） | 最早的记忆、家庭环境、最初的感受 |
| 学生时代 | 小学到大学、关键老师、同学、学校经历 |
| 初入社会 | 第一份工作、第一次独立、第一次重大失败 |
| 某段关系期 | 以一段重要关系为时间锚点的时期 |
| 某座城市期 | 以一座城市为空间锚点的时期 |
| 转变期 | 职业转换、身份变化、信念重塑的阶段 |
| 当下 | 正在经历、尚未成为"记忆"的内容 |

##### era 的特殊性

era 不同于其他维度——一段记忆可能同时属于多个时期（学生时代也是北京时期）。支持标记多个 era，但有一个"主要时期"。

#### 6. 第 3 维：emotional_weight（情感重量）

`type + era` 告诉你"什么形式、什么时候"，`emotional_weight` 告诉你"现在碰起来什么感觉"。

##### 4 级重量

| 重量 | 含义 | 接触时的反应 | 建议处理方式 |
|---|---|---|---|
| **轻** | 温暖的、好玩的、已经释然的 | 微笑、怀念、没有负担 | 随时可以翻阅，适合做回顾和分享 |
| **中性** | 事实性的，不带强烈情绪 | 平静、客观 | 作为人生档案的一部分 |
| **重** | 仍会触发情绪，但不至于无法承受 | 心里一紧、鼻子酸、需要缓一下 | 可以整理，但不要太频繁触碰 |
| **很重** | 尚未消化，触碰会很痛 | 想逃避、会哭、会情绪低落 | 先标记存在，不急着整理 |

##### emotional_weight 的动态性

情感重量是会变的。一条"很重"的记忆可能经过几年变成"重"再变成"中性"。这个变化本身就是成长的证据。

##### emotional_weight 不能被 processing 替代

- `emotional_weight` 回答："碰它的时候疼不疼？"（感受层面）
- `processing` 回答："我理解它了吗？"（认知层面）

一条记忆可以是"已整理"但"很重"——你理解了它，但想起来还是会难过。另一条可以是"暂不触碰"但"中性"——你不打算整理，但它并不让你疼。

#### 7. 记忆的评估属性：formative_power（塑造力）

> **重要前提**：formative_power **不是第 6 个分类维度**，它是穿透所有 5 维的**记忆内禀属性**。
> 5 维用来**归类**（按类型、时期、情感重量看分布），formative_power 用来**评估单条记忆**（它多大程度上造就了今天的我）。

##### 5 维 vs formative_power 的区分

| 5 个分类标签 | formative_power（评估属性） |
|---|---|
| **组织人生档案**：让你能问"童年时期我有哪些重要的地点记忆" | **判断单条记忆**：让你能问"这段经历对我的塑造有多深" |
| 出现在过滤器、分组、统计里 | 跟着每条记忆走，在记忆详情/人生回顾时显示 |
| 用来回答"发生了什么、什么时候、多重、整理好了吗" | 用来回答"我今天的样子有多少是从这里来的" |

##### 5 级取值

| 等级 | 含义 | 典型记忆 | 与 emotional_weight 的关系 |
|---|---|---|---|
| **极深** | 从根本上改变了自我认知、行为模式或人生方向 | 重大失去、决定性选择、关键领悟 | 可能是重的，也可能是已释然的 |
| **较深** | 显著影响了某个领域的看法或习惯 | 一段重要关系、一个生活阶段的结束 | 多为中等重量 |
| **中等** | 有一些影响，但不至于改变方向 | 一次有启发的旅行、一位短暂的老师 | 多为轻或中性 |
| **轻微** | 构成人生质地但几乎看不出直接因果 | 日常片段、某个下午的阳光 | 多为轻 |
| **无** | 纯粹的事实记录，没有塑造作用 | 几岁学会骑车、哪天搬了家 | 中性 |

##### 它能回答的问题

| 问题 | formative_power 怎么帮 |
|---|---|
| 我是怎么变成现在这样的？ | 过滤 formative_power=极深/较深，按 era 排列 |
| 哪些经历还在影响我的判断？ | 高 formative_power 的记忆 × 观念/原则模块的 source 字段交叉 |
| 我的人生转折点在哪里？ | 按时期看 formative_power 的峰值分布 |
| 我对自己的理解有多深？ | 高 formative_power 且有 processing=已整理 的比例 |

##### UI 呈现建议

- ✅ 记忆详情页**始终显示** formative_power
- ✅ 人生时间轴标注 formative_power 高的节点（加粗/放大）
- ✅ 可做"formative_power × emotional_weight"交叉——那些"极深且很重"的记忆是重点关注对象
- ❌ **不要**把 formative_power 放进主筛选器
- ❌ **不要**做"按塑造力排序的记忆列表"

#### 8. 第 4 维：processing（整理状态）

##### 7 种状态

| 状态 | 含义 | 操作建议 |
|---|---|---|
| **已整理** | 已经写清楚、理解过、安放好的 | 定期回顾，不频繁触碰 |
| **正在理解** | 正在写、正在想、正在消化中 | 给空间继续写，不催 |
| **暂不触碰** | 知道有这件事，但现在不适合打开 | 标记存在即可，完全不催 |
| **决定不再细究** | 承认它发生过，但选择不再回去翻 | 封存，尊重自己的选择 |
| **开放问题** | 想在未来搞清楚，但当前没有答案 | 保持开放状态，偶尔回看 |
| **想留给某人** | 希望未来被特定的人看到 | 标记接收者和条件 |
| **记不清的裂缝** | 知道有事情但细节模糊 | 接受不完整，不强迫补全 |

这一层背后的逻辑：

- 不是所有过去都需要被"理解"，有些只需要被"承认"
- 给沉重记忆一个"暂不触碰"的位置，比逼用户写出完整故事更重要
- 给"记不清"和"不想再说"留下尊严

#### 9. 第 5 维：privacy（隐私级别）

记忆是整个产品里最敏感的数据。privacy 维度让用户对每一条记忆的控制权清晰可见。

##### 5 级隐私

| 级别 | 含义 | 行为 |
|---|---|---|
| **仅自己** | 默认级别，只有自己可见 | 不出现在任何分享、导出、汇总视图中 |
| **需二次确认** | 即使是自己，查看时也需要再确认一次 | 打开时有确认步骤 |
| **指定的人** | 只对几个特定的人可见 | 在"告别与托付"场景中按指定列表交付 |
| **未来可公开** | 现在不公开，但以后某个时间点可以 | 设置解锁条件（如时间、事件） |
| **离世后可看** | 生前不公开，离世后按意愿交付 | 需要配套的交付机制 |

##### privacy 与 processing 的配合

有些 privacy=需二次确认 的记忆同时也是 processing=暂不触碰——双重保护。这种组合本身就是重要的信号：这段记忆需要被非常小心地对待。

#### 10. 记忆的录入：多条路径，不依赖"主动想起"

记忆和反思不一样，它常常不是被主动写下来的，而是被某样东西唤起来的。

##### 主动录入

- 自己想到了，新建一条
- 允许先标一个空壳（标题 + 时期），以后再补
- 允许只写一个地点、一个名字、一张照片，剩下留白

##### 从其他模块沉淀

- 一次反思里发现某件事是更早之前的延续 → 转化为记忆
- 一段关系记录里的事件 → 同时挂到记忆时间线上
- 一次情绪波动追溯到的源头 → 作为人生节点保留

##### 被感官线索唤起

- 上传一张照片作为锚点入口
- 标记一个地点、一件物品
- 这些锚点可以独立存在，不必先有事件

##### 被时间唤起

- 日期到了，模块自然带出"今天是某件事的 N 周年"
- 看到某段时期的旧记录时，提醒"那时候你写过这些"

##### 被别人唤起

- 别人提起一件事，自己想起但没准备好整理
- 先标记存在，比马上写清楚更重要

#### 11. 成长追踪：从记忆到成长

成长不是一条独立的记忆，而是**记忆集合显现的模式**。

##### 成长节点

当多条记忆指向同一个方向的变化时，就形成了一个"成长节点"：

- 从"不敢拒绝"到"能说不了" → 是一个成长节点
- 从"觉得都是自己的错"到"能看清对方的责任" → 是一个成长节点
- 从"害怕独处"到"能享受独处" → 是一个成长节点

成长节点的数据结构：
- 关联的多条"变化前"和"变化后"的记忆
- 触发转变的关键事件
- 这个成长属于哪个领域（关系 / 自我 / 工作 / 情绪能力 / 生活方式）
- 这个成长是否稳固（偶尔还会退回去 / 基本稳定 / 已经完全内化）

##### 成长与 formulative_power 的关系

通常 formulative_power 最高的那些记忆，就是成长节点的关键转折。反过来看，如果一个成长节点没有对应的 formulative_power 高的记忆，可能意味着那次转变的真正原因还没被找到。

#### 12. 边界约定

##### 决策规则

> 先问"**这段记忆以什么形式被记住**"（type），再问"**它属于哪段人生**"（era），最后问"**我现在怎么看待它**"（emotional_weight + processing）。

##### 记忆 vs 反思的区分

| | 记忆 | 反思 |
|---|---|---|
| **时间指向** | 过去 | 当下/最近 |
| **内容** | 曾经发生的事、地点、人、物 | 最近的想法、情绪、选择、困惑 |
| **目的** | 整理过去、理解来路 | 观察现在、调整方向 |
| **典型句式** | "那时候……""记得有一次……" | "最近我发现……""我在想……" |

一条反思如果反复出现、追溯到了很早之前 → 可能应该沉淀为一条记忆。

##### 常见归属疑问

| 内容 | 归属 | 理由 |
|---|---|---|
| 上月的一次重要对话 | 反思 | 太近，还没沉淀成"记忆" |
| 三年前同样的对话模式 | 记忆 | 已经足够远，可以回看 |
| 童年的家庭氛围 | 记忆（地点/事件混合） | 以空间为锚点的时期记忆 |
| "我最近发现我总是在逃避冲突" | 反思 | 当下的观察 |
| "我从小就不敢表达愤怒" | 记忆 | 对过去的整理，可以关联到具体经历 |

#### 13. 典型问题 → 怎么用回答

| 你的真实困惑 | 用哪几个维度 / 属性 |
|---|---|
| "我是怎么变成今天这样的？" | 过滤 formative_power=极深/较深，按 era 排列 |
| "还有哪些记忆压在我心里？" | 过滤 emotional_weight=重/很重 + processing≠已整理 |
| "童年对我影响有多大？" | 过滤 era=童年 + formative_power，看分布 |
| "有什么记忆我一直在逃避？" | 过滤 processing=暂不触碰 + emotional_weight=很重 |
| "我的人生转折点有几个？" | 看 formative_power=极深 的记忆在时间轴上的分布 |
| "哪段时期最艰难也最重要？" | 按 era 看 emotional_weight=很重 + formative_power=极深 的密度 |
| "我真正成长了哪些地方？" | 看成长节点列表，按领域分组 |
| "那些对我很重要的人留下了什么？" | 过滤 type=人物 + formative_power，看影响 |
| "什么地方承载了我最多的回忆？" | 过滤 type=地点，按 emotional_weight 看 |
| "我准备好面对哪些记忆了？" | 看 processing=正在理解 的记忆，这是当前最重要的整理线索 |

#### 14. 实施步骤（给开发者）

1. 把 6 种记忆类型作为 `MemoryType` 枚举写出来
2. 把 7 种整理状态作为 `ProcessingStatus` 枚举写出来
3. 把 5 级隐私作为 `PrivacyLevel` 枚举写出来
4. 把 5 级塑造力作为 `FormativePower` 枚举写出来
5. era 使用用户自定义标签 + 默认提示（不硬编码枚举）
6. 在 `Memory` 类型上：
   - `type: MemoryType`（必填）
   - `era: string[]`（必填，支持多个时期标签）
   - `emotionalWeight: EmotionalWeight`（必填）
   - `processing: ProcessingStatus`（必填）
   - `privacy: PrivacyLevel`（必填，默认"仅自己"）
   - `formativePower?: FormativePower`（可选，因为刚录入时可能不确定）
7. `GrowthNode` 类型：关联多条记忆（变化前/变化后）、领域、稳固程度
8. 记忆支持从反思/关系/情绪模块生成（跨模块沉淀）
9. privacy=需二次确认 的记忆在打开时触发确认步骤
10. UI 层：**5 维**进入过滤器/分组；**formative_power** 进入时间轴标注和详情页

#### 15. 与生命整理模块的关系

成长记忆模块和 [生命整理模块](../life-organization/design.md) 有天然的联系：

- 成长记忆关注"过去如何塑造了我"
- 生命整理关注"我想留下什么、如何告别"
- 一些记忆同时属于两个模块——比如"想留给某人的记忆"（processing=想留给某人）同时也是生命整理的内容
- 建议通过 privacy 和 processing 字段的交叉来实现联动，而不是把两个模块合并
