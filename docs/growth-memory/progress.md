- `development.md` 已收纳当前能从设计文档中提取的开发约定。

## 已完成

- 已按 `design.md` 的 5 个分类维度落地成长记忆页面：
  - `type`
  - `era`
  - `emotionalWeight`
  - `processing`
  - `privacy`
- 已提供页面级五维过滤器；`era` 保持用户自定义标签 / 默认提示，不做硬编码枚举。
- 已将 `formativePower` 保持为单条记忆详情、时间线标注和统计提示，不进入主筛选器。
- 已展示成长节点，并通过 `beforeMemoryIds / afterMemoryIds / triggerMemoryId` 关联记忆。
- 已展示感官锚点、回看问题和仍在生效的影响线索。
- 已补充固定壳一页式布局与移动端堆叠布局，参考购物 / 饮食模块的内部滚动与响应式方式。
- 已将顶部 Hero 收敛为纯统计栏，避免与公共 `PageIntro` 重复展示标题和说明。
- 已接入全局控制模式：
  - 浏览模式展示过滤、时间线、成长节点和辅助线索。
  - 管理模式展示结构检查、需二次确认数量、未评估塑造力条目和缺失记忆引用。
  - 管理模式支持新增、编辑、删除记忆、成长节点、感官锚点、回看问题和仍在生效的影响线索。
- 已补齐成长记忆页面中英文国际化，并补充公共搜索提示和工具栏管理模式等 label 的国际化。
- 已补齐 `journey.enum.*` 中英文枚举词典，记忆类型、情感重量、整理状态、隐私级别、塑造力、来源模块、成长领域和稳固程度在英文模式下不再回退显示中文原始枚举值。
- 已让 live workspace snapshot 中的 `growth` / `memory` 使用后端接口数据，不再在真实模式下用成长记忆 mock 兜底。
- 已新增 `save_growth` / `save_memory` Tauri command，并接入 live API、mock API 与 `useSaveJourneyMutation`，保存后刷新 workspace snapshot / journey query。
- 已保留 mock 模式的模拟数据，并按文档补充更多样本：
  - 覆盖 `想留给某人`、`记不清的裂缝` 等整理状态。
  - 补充新的地点 / 物件 / 事件 / 领悟记忆。
  - 补充新的成长节点、感官锚点和影响线索。
