## 已完成

- 按 `design.md` 的 5 个视图落地情绪页：`情绪总览`、`今天的情绪记录`、`波动时间线`、`触发因素与关联`、`安抚与恢复工具箱`。
- 参考购物/饮食模块的控制模式：浏览模式只展示数据，控制模式显示新增、编辑、删除入口。
- 页面不再依赖组件内临时模拟数据；live 模式通过 Tauri 后端读取/保存 `emotion.json`。
- 已删除前端情绪模块旧 mock 数据文件，前端 mock snapshot 仅保留空情绪 fallback，避免与后端样本形成两套来源。
- 新增后端 `src-tauri/src/emotion/seed.json`，覆盖记录、趋势、触发、生活关联、环境/关系线索、恢复工具、无效动作和恢复经验等初始样本数据。
- 新增 `get_emotion` / `save_emotion` Tauri command，并接入 live API、mock API 与 React Query 保存 mutation。
- 编辑器覆盖当前页面展示的数据块：总览、情绪记录、趋势点、时间段、循环模式、触发因素、生活关联、环境提示、关系提示、恢复工具、极简步骤、无效动作和恢复经验。
- 补齐生活关联、环境提示、关系提示、极简步骤、无效动作和恢复经验等子列表的空态，控制模式下空列表仍保留可新增或可编辑入口。
- 补充 `emotion` 国际化命名空间，覆盖页面文案、操作按钮、编辑器字段与情绪相关枚举。
- 情绪编辑弹窗实体标题的 fallback 已从中文拼接改为英文中性兜底；正常展示仍使用 `emotion.editor.*.createTitle/editTitle` 中英文词典，避免缺 key 时英文界面露出中文。
- 在三套工作区主题中补充情绪模块语义变量：`--emotion-accent-*` 与 `--emotion-chart-line`。
