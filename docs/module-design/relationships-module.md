# 关系深化模块 - 关系分类与观察建议

## 1. 30 秒看懂这份文档

这份文档建议给每段重要关系打 **5 个分类标签** + **1 个评估属性**：

### 5 个分类标签（用来归类、过滤、看关系世界）

| 标签 | 它回答什么 | 取值示例 |
|---|---|---|
| **type**（关系类型） | 这个人是谁？ | 家人 / 伴侣 / 朋友 / 同事 / 过去重要的人 / 导师（共 6 个，见 §4） |
| **depth**（深度） | 这段关系有多近？ | 亲密 / 亲近 / 熟人 / 疏远 / 断联 |
| **stage**（阶段） | 这段关系现在处于什么阶段？ | 建立中 / 稳定 / 紧张 / 修复中 / 已结束 / 等待中 |
| **impact**（影响方向） | 这段关系在滋养还是消耗？ | 滋养 / 消耗 / 中性 / 混合 |
| **interaction**（互动频率） | 多久会联系一次？ | 每天 / 每周 / 每月 / 每年 / 几乎不 / 已无联系 |

### 1 个评估属性（跟着每段关系走，用来衡量"这一段"）

| 属性 | 它回答什么 | 取值示例 |
|---|---|---|
| **unfinished_weight**（未完成重量） | 这段关系有多少没说完的话、没解开的结？ | 很重 / 中等 / 轻微 / 无 |

> **5 维和未完成重量的区别**：5 维用来**归类**（"哪些关系在消耗我"）；未完成重量用来**评估**（"这段关系我没说完的话有多重"）。详见 §7。

打满这 5 个标签 + 1 个属性后，原来问不出的问题就能问出来了：

- "我的关系世界里谁在滋养我、谁在消耗我？"
- "哪些关系正在慢慢疏远，哪些需要修复？"
- "我在关系里反复出现什么模式？"
- "哪些关系有没说完的话，压在我心里？"

## 2. 为什么不按"人和我的关系"简单分类

如果只按关系名称分类，会出现这样的列表：

> 妈妈 / 爸爸 / 伴侣 / 大学同学 A / 前同事 B / 发小 C / …

这种分类的问题：

1. **看不出关系质量**——"妈妈"是一个标签，但你和妈妈之间是滋养还是消耗，看不出来
2. **发现不了全局模式**——你可能有 5 个"消耗型"关系，但分散在不同类型里，看不见
3. **追不到变化**——你和某个朋友从每周联系变成了每年联系，静态标签无法体现
4. **驱动不了行动**——知道"我有 3 个前同事好友"，不知道"哪段关系现在最需要修复"

### 换一个问法

不要问"这个人是谁"，问"**这段关系现在处于什么状态？它对我产生了什么影响？**"

> **什么叫"关系阶段"**——关系不是静止的角色标签。同一个人，可能在某个时期是滋养的，另一个时期是消耗的；可能曾经亲密后来疏远，也可能正在从紧张走向修复。阶段和影响方向会变化，而类型相对稳定。

### 按这套维度分类的好处

- **看全局**：一眼看到关系世界里有几个"滋养"、几个"消耗"
- **看趋势**：stage 的变化本身就是关系演变的轨迹
- **看模式**：按 type + impact + depth 交叉，能发现"我总是在 XX 类型的关系里消耗自己"
- **对行动有价值**：哪些关系需要修复、哪些需要拉开距离、哪些要珍惜，有数据支撑

## 3. 用两段具体关系走一遍

### 例 1：一段正在疏远的多年朋友关系

**5 个分类标签**：

| 维度 | 取值 | 为什么 |
|---|---|---|
| type | **朋友** | 认识多年，没有血缘和职场关系 |
| depth | **亲近→疏远** | 曾经很亲近，现在联系越来越少 |
| stage | **紧张** | 最近有过一次不愉快的沉默，还没说开 |
| impact | **混合** | 见面时还是开心的，但不见的时候心里有疙瘩 |
| interaction | **每月→几乎不** | 从每月约变成了偶尔点赞 |

**评估属性**：

| 属性 | 取值 | 含义 |
|---|---|---|
| unfinished_weight | **中等** | 有些话想说但不知道怎么开口，不解决的话会一直挂着 |

> 现在你可以问：我所有**混合/消耗**（impact）的**朋友关系**（type）里，**unfinished_weight > 轻微** 的有哪几个？这就是需要处理的清单。

### 例 2：一段稳定的伴侣关系

**5 个分类标签**：

| 维度 | 取值 | 为什么 |
|---|---|---|
| type | **伴侣** | 亲密关系的核心 |
| depth | **亲密** | 彼此深度了解，可以袒露脆弱 |
| stage | **稳定** | 已经过了磨合期，现在彼此舒服 |
| impact | **滋养** | 这段关系让人更平静、更有力量 |
| interaction | **每天** | 每天都在一起或联系 |

**评估属性**：

| 属性 | 取值 | 含义 |
|---|---|---|
| unfinished_weight | **轻微** | 有些小摩擦但当场说了，没积累 |

> 两段关系类型不同，阶段不同，影响不同，未完成重量也完全不同——这就是 **5 维互不替代**。

## 4. 第 1 维：6 种关系类型

### 为什么是 6 种

按**关系的本质纽带**来分，而不是按社交标签。

| 类型 | 纽带本质 | 典型角色 | 看完整性的关键问题 |
|---|---|---|---|
| **家人** | 血缘或成长绑定 | 父母、兄弟姐妹、子女、养育人 | "原生关系在我生命里留下了什么？" |
| **伴侣** | 浪漫/情感深度绑定 | 恋人、配偶 | "我能在一个人面前完全做自己吗？" |
| **朋友** | 选择性的情感连接 | 好友、发小、知己、玩伴 | "我主动选择的关系质量如何？" |
| **同事** | 工作/职业场景绑定 | 同事、上下级、合作伙伴 | "职场关系在帮我还是在消耗我？" |
| **过去重要的人** | 曾经深度绑定但已变化 | 前任、旧友、已故亲人 | "离开的人还在怎样影响我？" |
| **导师/榜样** | 单向或双向的引领关系 | 老师、mentor、崇敬的人 | "谁在指引我的方向？" |

### 类型的交叉

一个人可以同时属于两个类型吗？可以，但选**当前最主要的互动场景**。

- 同事后来成了朋友 → 最主要的互动场景 是朋友（私下的情感连接为主），归为**朋友**
- 伴侣也是最好的朋友 → 纽带本质是伴侣，归为**伴侣**；"朋友"的侧面在 depth 里体现

## 5. 第 2 维：depth（深度）

`type` 回答"我们的纽带是什么"，`depth` 回答"我在这段关系里有多敞开"。

### 5 级深度

| 深度 | 含义 | 典型表现 |
|---|---|---|
| **亲密** | 可以袒露脆弱、分享最真实的想法 | 不需要筛选，可以直接说"我今天很崩" |
| **亲近** | 有信任和温暖，但不会全部敞开 | 聊天舒服，但有些事不会主动提 |
| **熟人** | 认识、有交集，但情感连接弱 | 见面能聊，不见不会想起 |
| **疏远** | 曾经近过，现在距离拉大 | 心里还有这个人的位置，但生活里几乎没有 |
| **断联** | 完全不再联系 | 可能是不想联系，也可能是客观原因 |

### depth 的动态性

depth 是所有维度里变化最频繁的。一段关系可以在几个月内从"亲近"滑到"疏远"，也可以在修复后回升。depth 的变化记录本身就是关系时间线的核心线索。

## 6. 第 3 维：stage（阶段）

`type + depth` 回答了"是什么、多近"，`stage` 回答"它现在在往哪个方向走"。

### 6 种阶段

| 阶段 | 含义 | 典型信号 |
|---|---|---|
| **建立中** | 正在靠近、了解、建立信任 | 还不太了解对方，但想继续接触 |
| **稳定** | 关系节奏平稳，没有明显动荡 | 不需要频繁维护也能保持连接 |
| **紧张** | 有未解决的摩擦或正在冲突中 | 说话小心翼翼、回避某些话题 |
| **修复中** | 经历过破裂或疏远，正在尝试恢复 | 主动沟通、道歉、调整相处方式 |
| **已结束** | 明确结束了，无论是和平还是痛苦 | 不再联系、或已经告别 |
| **等待中** | 没有结束，但暂时没有推进 | 对方没回应、自己也在犹豫要不要主动 |

### stage 不能被 impact 替代

- `stage` 回答："关系现在在什么状态？"——描述性的
- `impact` 回答："这段关系对我好不好？"——评价性的

一段"稳定"的关系可能是"消耗"的（习惯性忍受），一段"已结束"的关系可能是"滋养"的（离开后反而释然了）。分开看才能发现反常组合。

## 7. 关系的评估属性：unfinished_weight（未完成重量）

> **重要前提**：unfinished_weight **不是第 6 个分类维度**，它是穿透所有 5 维的**关系内禀属性**。
> 5 维用来**归类**（按类型聚合、按深度看亲近度、按阶段看趋势），unfinished_weight 用来**评估单段关系**（它在我心里还挂着多少没完结的东西）。

### 5 维 vs unfinished_weight 的区分

| 5 个分类标签 | unfinished_weight（评估属性） |
|---|---|
| **组织关系世界**：让你能问"哪些朋友关系在消耗我" | **判断单段关系**：让你能问"这段关系我还有什么没说完的话" |
| 出现在过滤器、分组、统计里 | 跟着每段关系走，在关系详情/想说的话页面显示 |
| 用来回答"谁在滋养我、消耗我、多久没联系了" | 用来回答"我心里还压着什么、哪段关系该去收尾了" |

### 4 级取值

| 等级 | 含义 | 典型信号 | 建议动作 |
|---|---|---|---|
| **很重** | 有非说不可但还没说的话，压在心里 | 反复想起这个人、有情绪波动、梦见 | 优先处理——写下来或说出来 |
| **中等** | 有些话想说，但还没到非说不可 | 偶尔想起，心里咯噔一下 | 先写下来，不一定发 |
| **轻微** | 有一些小遗憾或没完全说清的地方 | 偶尔想起但不影响日常 | 可以放着，也可以顺手解决 |
| **无** | 没有挂碍，关系已经完结或完全通畅 | 想起时心里是平静的 | 保持即可 |

### 适用边界

不是每段关系都有未完成重量。字段设为 optional，逻辑上：

- ✅ `stage ≠ 稳定` 且 `depth 曾经更深` → 通常有 unfinished_weight
- ✅ `stage = 已结束` → 取决于结束得是否干净
- ❌ 刚认识的关系（`stage = 建立中`）→ 几乎没有
- ❌ 轻松无压力的浅层关系 → 几乎没有

### 它能回答的问题

| 问题 | unfinished_weight 怎么帮 |
|---|---|
| 哪些关系我该去收尾了？ | 过滤 unfinished_weight=很重，优先处理 |
| 我为什么总是想起这个人？ | 看是否 unfinished_weight 很高但没有被处理 |
| 哪段关系的结束最不干净？ | stage=已结束 + unfinished_weight=很重/中等 |
| 我是不是在逃避一些重要的对话？ | 看高 unfinished_weight 的关系里有多少"想说的话"还是空的 |

### UI 呈现建议

- ✅ 关系详情页**始终显示** unfinished_weight
- ✅ "想说的话"页面按 unfinished_weight 排序，重的排前面
- ✅ 关系总览可显示"未完成重量"的分布（几段关系还挂着）
- ❌ **不要**把 unfinished_weight 放进主筛选器
- ❌ **不要**做"按未完成重量分类的关系列表"

## 8. 关系条目内的子结构

### 每段关系应该承接的内容

| 内容 | 说明 |
|---|---|
| **关系档案** | 这个人是谁，在自己生命中的位置，整体感受关键词 |
| **关键互动事件** | 第一次认识、重要谈话、冲突、和好、疏远、断联、重逢 |
| **情绪线索** | 信任、依赖、愧疚、愤怒、想念、亏欠、安全感 |
| **想说的话** | 感谢、道歉、解释、告别、坦白、祝福 |
| **关系带给自己的** | 正面影响（被教会了什么）和持续阴影（还在影响什么） |
| **边界状态** | 目前有没有清晰的边界？有没有被反复突破的地方？ |

### 关系模式复盘（跨关系视图）

从单段关系的数据里跳出，看跨关系的重复模式：

- 自己在关系中反复扮演的角色（讨好、退缩、拯救、逃避、过度负责）
- 容易被哪类人吸引
- 最常触发自己的关系情境
- 经常出现的冲突路径
- 边界在哪里最容易变弱

## 9. 边界约定

### 决策规则

> 先问"**我和这个人的纽带本质是什么**"，再问"**当前最主要的互动场景是什么**"。
> type 选纽带本质，depth/stage/impact 反映当前状态。

### 常见归属疑问

| 关系 | type 归属 | 理由 |
|---|---|---|
| 既是同事又是朋友 | 朋友 | 当前最主要互动是私下情感连接 |
| 前任 | 过去重要的人 | 纽带已经变化，但影响还在 |
| 已故亲人 | 过去重要的人 | type 可同时标记"家人"但主要归入"过去重要的人" |
| 网友 / 游戏队友 | 朋友 | 选择性的情感连接，不一定要见面 |
| 心理咨询师 | 同事（职业关系） | 职业场景绑定，但深度可能很高 |
| 老师 | 导师/榜样 | 引领关系 |
| 室友 | 看情况 | 有情感连接→朋友；纯粹合租→熟人/中性 |

### 关系与"想说的话"页面

想说的话不一定只写给现有关系。写给"无法再联系的人""已故的人""未来的自己"的话，也需要承接——但它们不一定是独立的关系条目。建议在"想说的话"页面支持三种目标：

1. 写给现有关系条目里的人
2. 写给不在关系列表中的人（不创建关系条目，只保留文字）
3. 写给未来的自己或某个还没出现的人

## 10. 典型问题 → 怎么用回答

| 你的真实困惑 | 用哪几个维度 / 属性 |
|---|---|
| "谁在滋养我，谁在消耗我？" | 按 impact 分组，直接看两边的名单 |
| "哪段关系该去修复了？" | 过滤 stage=紧张/修复中 + impact=曾经滋养 |
| "哪段关系该拉开了？" | 过滤 impact=消耗 + stage=稳定（习惯性忍受） |
| "我是不是在逃避一些对话？" | 看 unfinished_weight=很重 的列表 |
| "我的关系世界里朋友多还是消耗多？" | 按 type + impact 交叉统计 |
| "哪段关系最近在变坏？" | 看 stage 最近从稳定→紧张 的变化 |
| "有没有人我该重新联系？" | 过滤 depth=疏远 + unfinished_weight=中等/很重 |
| "我在关系里反复出什么问题？" | 关系模式复盘——跨关系的高频角色和冲突路径 |
| "这段关系到底带给我什么？" | 单段关系的"正面影响""持续阴影"字段 |
| "有没有什么话我该在来得及的时候说？" | unfinished_weight=很重 的关系 + "想说的话"是否还是空的 |

## 11. 实施步骤（给开发者）

1. 把 6 种关系类型作为 `RelationshipType` 枚举写出来
2. 把 5 级深度作为 `RelationshipDepth` 枚举写出来
3. 把 6 种阶段作为 `RelationshipStage` 枚举写出来
4. 把 4 种影响方向作为 `RelationshipImpact` 枚举写出来
5. 把 6 级互动频率作为 `InteractionFrequency` 枚举写出来
6. 把 4 级未完成重量作为 `UnfinishedWeight` 枚举写出来
7. 在 `Relationship` 类型上：
   - `type: RelationshipType`（必填）
   - `depth: RelationshipDepth`（必填）
   - `stage: RelationshipStage`（必填）
   - `impact: RelationshipImpact`（必填）
   - `interaction: InteractionFrequency`（必填）
   - `unfinishedWeight?: UnfinishedWeight`（可选）
8. 关系条目内挂载：关键互动事件（子列表）、情绪线索（标签）、想说的话（子列表）
9. depth 和 stage 保留变化历史（时间线数据来源）
10. "想说的话"支持三种目标类型：关系条目内的人 / 独立对象 / 未来的自己
11. UI 层注意：**5 维**进入过滤器 / 分组 / 总览；**unfinished_weight** 进关系详情和"想说的话"排序

## 12. 不在本文档讨论的范围

- 关系条目的录入、编辑交互
- 数据库表结构
- 关系与情绪/记忆/观念模块之间的具体联动逻辑
- 隐私控制的技术实现（谁可以看到关系数据）
