# 社会经济模块 - 外部经济常识分类建议（草稿）

> 这是一份**草稿**，目的是先把分类骨架定下来。细节示例和边界情况后续再补。

## 0. 这个模块的定位

不同于已有的 7 个模块（观念、原则、关系、成长记忆、营养、生活组织、购物），这是**第一个"向外"的模块**——它储存的不是"我怎么运转"，而是**"外部经济世界怎么运转"**。

所以它不适合套用观念/原则的"自我评估"维度（`impact=支撑性/限制性`、`cost=高代价/低代价`），因为通胀本身没有"在帮我还是限我"。

这个模块的核心问题不是"我信什么"，而是：

- **我对外部经济世界的认知地图，覆盖到哪了？**
- **哪些块我自以为懂、其实只是听过名词？**
- **哪些知识离我的决策很近，哪些只是认知储备？**

## 1. 30 秒看懂这份文档

每一条社会经济知识打 **4 个分类标签** + **1 个评估属性**：

### 4 个分类标签

| 标签 | 它回答什么 | 取值示例 |
|---|---|---|
| **domain**（领域） | 这条知识属于经济运行的哪一块？ | 货币与物价 / 个人财务 / 劳动力市场 / 产业与公司 / 财政与政策 / 金融市场 / 全球与宏观（共 7 个，见 §4） |
| **layer**（层次） | 它是在哪一层的视角？ | 微观（个人/家庭） / 中观（行业/市场） / 宏观（国家/全球） |
| **confidence**（掌握程度） | 我对这条理解到什么程度？ | 听过名词 / 知道大致逻辑 / 能预判常见情境 / 有自己的判断框架 |
| **source**（来源） | 这条认知从哪来的？ | 系统学习 / 新闻媒体 / 亲身经历 / 他人叙述 / 专业讨论 |

### 1 个评估属性（跟着每条知识走）

| 属性 | 它回答什么 | 取值示例 |
|---|---|---|
| **relevance**（决策距离） | 这条知识跟我的决策有多近？ | 直接影响当前决策 / 影响中期规划 / 影响长期方向 / 纯认知储备 |

> **4 维和决策距离的区别**：4 维用来**归类**（"关于劳动力市场我懂什么"）；决策距离用来**评估**（"这条知识现在用得上吗")。详见 §7。

打满这套标签后能回答：

- "我的经济常识地图，哪块是空的？"
- "我以为我懂、但其实只是听过名词的，有哪些？"
- "我最近的决策（买房 / 换工作 / 投资），相关知识我覆盖到了吗？"
- "我对宏观经济的理解，是看新闻拼起来的，还是真有框架？"

## 2. 为什么不按"经济学教科书章节"分类

如果按教科书分（微观经济学、宏观经济学、国际经济学、金融学……），会出现的问题：

1. **跟决策不挂钩**——知道"边际效用递减"在教科书第 3 章，但不知道它在生活里管什么
2. **看不出掌握程度**——一个名词在脑子里是"听过"还是"能用"，差别巨大，但教科书分类掩盖了这个差别
3. **看不出认知盲区**——按教科书分会显得"都覆盖了"，但实际上每一章只懂了 5%
4. **看不见知识在动**——通胀机制、利率传导这些东西在不同经济周期表现不同，固定的教科书结构表达不出"我对 X 的理解还在更新"

### 换一个问法

不要问"这条知识在教科书哪一章"，问"**它属于哪个生活相关的经济领域？它是个人层面的还是宏观层面的？我对它掌握到什么程度？**"

## 3. 用两条具体知识走一遍

### 例 1："央行加息会让股市承压，但传导有时滞"

| 维度 | 取值 | 为什么 |
|---|---|---|
| domain | **金融市场** | 核心是关于股票市场的反应机制 |
| layer | **宏观** | 它是国家货币政策层面的传导 |
| confidence | **知道大致逻辑** | 能说出"加息→资金成本上升→估值压力"，但具体时滞多久、哪些行业先反应说不清 |
| source | **新闻媒体 + 专业讨论** | 主要从财经新闻和播客拼起来的 |
| **relevance** | **影响中期规划** | 影响我对资产配置节奏的判断，但不是今天就要做的决定 |

### 例 2："通胀对持有现金的人是隐性税"

| 维度 | 取值 | 为什么 |
|---|---|---|
| domain | **货币与物价** | 核心是关于货币购买力的变化 |
| layer | **微观** | 它在描述这件事如何作用于一个家庭/个人 |
| confidence | **能预判常见情境** | 能识别"什么情况下持有现金亏得最多"、"通胀和资产价格的非对称影响" |
| source | **系统学习 + 亲身经历** | 看过相关书籍，也经历过几年物价上涨 |
| **relevance** | **直接影响当前决策** | 影响我现在的储蓄/资产配置选择 |

> 两条知识 domain 不同、layer 不同、confidence 也不同——这就是 **4 维互不替代**加上**决策距离穿透 4 维**的体现。

## 4. 第 1 维：7 个领域（暂定）

> 这是骨架的核心争议点，可能需要你看了之后调整。

| 领域 | 它覆盖什么 | 典型知识点 |
|---|---|---|
| **货币与物价** | 通胀、通缩、汇率、购买力、利率的本质 | "通胀是怎么形成的""为什么央行调利率" |
| **个人财务** | 储蓄、负债、资产配置、保险、退休规划 | "复利效应""债务的杠杆与风险""资产配置的基本逻辑" |
| **劳动力市场** | 就业、薪资、行业兴衰、职业经济 | "为什么某些行业薪资在涨""学历的回报曲线""自由职业的经济学" |
| **产业与公司** | 公司怎么赚钱、行业逻辑、商业模式 | "平台经济的网络效应""一家公司的护城河是什么" |
| **财政与政策** | 政府税收、转移支付、补贴、公共支出 | "税制怎么影响普通人""政府支出的优先级如何决定" |
| **金融市场** | 股票、债券、利率、汇率市场的运作 | "估值的基本逻辑""债券价格和利率的反向关系" |
| **全球与宏观** | 国际贸易、地缘对经济的影响、全球资本流动 | "汇率为什么会变""产业链转移的逻辑" |

### 没单独切的话题（合并到上面）

- **房地产** → 跨在"个人财务"（自住）、"金融市场"（投资属性）、"财政与政策"（土地财政）。不单独成域，按主要视角归。
- **社会保障** → 归到"财政与政策"。
- **创业 / 中小企业经济** → 归到"产业与公司"。

> 如果你觉得房地产、社会保障这种话题足够大、值得单独成域，告诉我，我再调整。

## 5. 第 2 维：layer（视角层次）

| 层次 | 它回答 | 示例 |
|---|---|---|
| **微观** | 这件事如何作用于个人/家庭 | "通胀如何影响我家的购买力""加息如何影响我的房贷" |
| **中观** | 这件事在行业/市场层面怎么运作 | "互联网行业为什么在收缩""新能源车的成本结构" |
| **宏观** | 这件事在国家/全球层面的运转 | "央行怎么决定利率""中美贸易摩擦的传导机制" |

### 同一个话题在三层有不同切片

比如"加息"：

- 微观："我的房贷月供会涨多少"
- 中观："哪些行业会先受压"
- 宏观："央行为什么这个时间点加息"

三层都懂，才算真的理解了"加息"。**layer 维度让这种"懂的层次"可见**——你可能在微观层很懂、宏观层完全空白。

## 6. 第 3 维：confidence（掌握程度）

这是这个模块**最关键、最区别于其他模块**的维度。

| 等级 | 含义 | 典型表现 |
|---|---|---|
| **听过名词** | 知道存在这个概念，说不清机制 | "我知道'量化宽松'这个词，但具体是怎么操作的说不清" |
| **知道大致逻辑** | 能给朋友解释主要的因果链 | "QE 就是央行印钱买债券，把利率压下去刺激经济" |
| **能预判常见情境** | 看到现象能推机制、看到机制能预判后果 | "现在美联储 QE，所以新兴市场资本会怎么动，资产价格会怎么走" |
| **有自己的判断框架** | 能反向推论、能识别例外、能与其他知识结合 | "这次 QE 和 2008 那次的不同点是 X，所以传导会有 Y 的差异" |

### 为什么这一维比所有其他维都重要

经济学知识有个特点：**听过 ≠ 理解 ≠ 能用**。

- "听过通胀"的人和"能解释通胀机制的人"，对自己财务的判断完全不同
- "知道大致逻辑"和"有自己的判断框架"之间，差着无数次实际验证

这一维让"我以为我懂"和"我真的懂"的差距显形。是这个模块**最能驱动学习行动**的字段——`confidence=听过名词` 直接告诉你"这块该补了"。

## 7. 评估属性：relevance（决策距离）

> **重要前提**：relevance **不是第 5 个分类维度**，它是穿透所有 4 维的**单条知识的内禀属性**。

### 4 维 vs relevance 的区分

| 4 个分类标签 | relevance（评估属性） |
|---|---|
| **组织认知地图**：让你能问"宏观层面我懂哪些" | **判断单条知识**：让你能问"这条现在跟我的决策近不近" |
| 出现在过滤器、分组、统计里 | 跟着每条知识走，在详情页/决策时显示 |

### 4 级取值

| 等级 | 含义 | 典型场景 |
|---|---|---|
| **直接影响当前决策** | 现在/最近就要用 | "我正在考虑买房，房贷利率走势直接影响" |
| **影响中期规划** | 1-3 年内会用到 | "未来几年的资产配置、职业方向" |
| **影响长期方向** | 5 年以上的方向感 | "养老、子女教育、移民、行业大方向" |
| **纯认知储备** | 暂时用不上，但拓宽世界观 | "了解一下产业链转移的逻辑" |

### 它能回答的问题

| 问题 | relevance 怎么帮 |
|---|---|
| 我最近要做的大决定，相关知识储备够吗？ | 过滤 relevance=直接影响当前决策，看 confidence 是不是都到位 |
| 我学的东西是真的有用，还是只在拓宽认知？ | 看 relevance 分布——纯储备占比过高，说明跟生活脱节 |
| 我该优先补哪块？ | relevance=直接影响 + confidence=听过名词，是补课的第一优先级 |

### UI 呈现建议

- ✅ 详情页**始终显示** confidence 和 relevance（两个评估属性的核心）
- ✅ 总览可做"领域 × confidence"热力图（哪些领域我掌握薄弱）
- ✅ 决策入口可做"按 relevance 筛选 + 按 confidence 排序"的复习视图
- ❌ **不要**把 relevance 放进主筛选器（那是 4 维的位置）

## 8. 边界约定（草稿，待细化）

### 跟其他模块的关系

| 这个模块存的是 | 不要混进来的是 |
|---|---|
| 外部经济世界的运行机制（事实/规律/概念） | 我对钱的看法（→ 观念模块的"金钱"领域） |
| 我对这些机制的理解程度 | 我的财务原则（→ 原则模块的"金钱"领域） |
| 客观知识 + 我的掌握程度 | 我的财务状况、收支（这是另一个层级的实操，不在本模块） |

> 简单分法：**这个模块回答"世界怎么运转"，观念模块回答"我怎么看"，原则模块回答"我怎么做"**。

### 知识 vs 观念的边界

- "通胀让持有现金的人贬值" → 知识（社会经济模块）
- "钱够用就好，不必焦虑通胀" → 观念（观念模块的金钱领域）

两者关联：知识可以**支撑**或**冲击**观念。这种联动后续可以做跨模块关联。

## 9. 实施步骤（草稿）

1. 7 个领域 → `EconDomain` 枚举
2. 3 个层次 → `EconLayer` 枚举
3. 4 级掌握程度 → `EconConfidence` 枚举
4. 5 种来源 → `EconSource` 枚举
5. 4 级决策距离 → `EconRelevance` 枚举
6. `EconKnowledge` 类型上 5 个字段必填
7. 支持关联：相关概念、对立观点、上下游因果
8. 每条保留 confidence 的修订历史（"我什么时候从'听过名词'升到'知道逻辑'"，本身是重要的学习轨迹）

## 10. 还没想清楚 / 需要你定的问题

1. **领域切法**：7 个领域合不合适？特别是"房地产"要不要单独成域？
2. **模块命名**：用"社会经济"、"经济常识"、"经济认知"哪个更合适？文件名暂用 `socioeconomics-module.md`。
3. **知识 vs 笔记**：一条记录是只存"知识点 + 我的标签"，还是要额外有个"我对它的理解笔记"字段？后者更接近 Anki/Obsidian 的形态。
4. **跟新闻 / 阅读输入的关系**：要不要支持"从某篇文章/书摘录"的来源溯源？这会让 source 维度变成多对多。
5. **学习节奏**：要不要内建"复习提醒"（confidence=听过名词 的知识隔一段时间提醒补课）？这超出了分类设计，属于功能层。

---

> 这份骨架延续了观念/原则模块的 "N 维 + 1 评估属性" 结构，但用 `confidence + relevance` 替代了 `impact / cost`——因为这是外部知识，不是自我状态。你看完之后，如果方向 OK，我们再展开 §4 的领域细分、§3 的更多例子、以及 §10 的几个待定问题。
