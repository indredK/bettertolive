# 原则模块设计文档
> 汇总 原则模块 的模块定位、分类维度、页面/数据设计和边界。

---

## 原则模块设计

### 原则模块 - 个人决策准则分类建议

#### 1. 30 秒看懂这份文档

这份文档建议给每一条原则打 **5 个分类标签** + **1 个评估属性**：

##### 5 个分类标签（用来归类、过滤、看决策体系）

| 标签 | 它回答什么 | 取值示例 |
|---|---|---|
| **domain**（领域） | 它在哪个生活面起作用？ | 关系 / 工作 / 金钱 / 健康 / 时间 / 诚信（共 6 个，见 §4） |
| **type**（类型） | 它是什么性质的标准？ | 边界 / 标准 / 底线 |
| **strength**（强度） | 它有多不可退让？ | 不可退让 / 强烈偏好 / 参考指引 |
| **source**（来源） | 它是怎么形成的？ | 受伤后确立 / 观察他人 / 主动推导 / 家庭继承 |
| **status**（状态） | 它目前在什么阶段？ | 生效中 / 正在测试 / 已修订 / 已放弃 |

##### 1 个评估属性（跟着每条原则走，用来衡量"这一条"）

| 属性 | 它回答什么 | 取值示例 |
|---|---|---|
| **cost**（代价） | 守住这条原则要付出什么？ | 高代价 / 中等代价 / 低代价 / 零代价 |

> **5 维和代价的区别**：5 维用来**归类**（"关系领域我有哪些原则"）；代价用来**评估**（"守住这条底线我要付出什么"）。详见 §7。

打满这 5 个标签 + 1 个属性后，原来问不出的问题就能问出来了：

- "我的底线主要集中在哪些领域？"
- "哪些原则我已经守不住了？"
- "哪些原则其实是在保护旧伤口？"
- "我现在最容易被突破的是哪条边界？"

#### 2. 为什么不按"原则字面内容"分类

如果按原则的内容分类，会出现这样的列表：

> "不借钱给朋友" / "不加班到 10 点以后" / "不在生气时做决定" / "每月存 30%" / …

这种分类的问题：

1. **看不出保护对象**——"不借钱给朋友"和"不在生气时做决定"看似无关，其实都在保护关系的稳定性
2. **分不清轻重**——所有原则列在一起，"每月存 30%"和"不受人身侵犯"看起来像是同一级别
3. **发现不了矛盾**——"不加班到 10 点"和"工作优先"可能同时在生效，但互相打架
4. **看不清来源**——不知道哪些原则是"受伤后立的"，哪些是"别人告诉我的"

##### 换一个问法

不要问"这条原则说了什么"，问"**它在保护哪个领域？它是边界还是标准？它有多不可退让？**"

> **什么叫"类型"**——底线是你觉得"突破这一条我就不再是我了"；边界是"这一片区域你不能进来"；标准是"做这类事时我优先这样选"。强度接近但性质不同。
>
> 比如"不受人身侵犯"是底线，"加班不超过 10 点"是边界，"做决定前先睡一觉"是标准。

##### 按这套维度分类的好处

- **知道哪块最脆弱**：按 domain 分组后，发现"关系领域原则很多，但健康领域几乎没有"——意味着健康容易被牺牲
- **分清是防御还是建设**：source=受伤后确立 的原则多是防御性的，需要定期检查是否还适用
- **提前知道什么会踩线**：strength=不可退让 的原则一旦被触发，就是大事
- **为决策校准提供结构**：面对一个选择，能快速调出对应领域的生效原则

#### 3. 用两条具体原则走一遍

##### 例 1："不在深夜做重要决定"

**5 个分类标签**：

| 维度 | 取值 | 为什么 |
|---|---|---|
| domain | **时间** | 它保护的是"在什么状态下做决定" |
| type | **标准** | 它是一套操作流程，不是不许别人靠近的边界 |
| strength | **强烈偏好** | 大多数时候遵守，但在极端情况下可以破例 |
| source | **亲身经历** | 半夜做了几个错误决定后总结出来的 |
| status | **生效中** | 一直在用，且持续有效 |

**评估属性**：

| 属性 | 取值 | 含义 |
|---|---|---|
| cost | **低代价** | 只是改变做决定的时间，几乎没有实质损失 |

> 现在你可以问：**时间领域**（domain）里有哪些**生效中**（status）的原则？它们的**代价**（cost）高吗？有没有需要升级或降级的？

##### 例 2："不接受长期单向付出的关系"

**5 个分类标签**：

| 维度 | 取值 | 为什么 |
|---|---|---|
| domain | **关系** | 它保护关系中的对等性 |
| type | **底线** | 突破这一条会让自己丧失关系中的自我尊重 |
| strength | **不可退让** | 已经在旧关系里退让太多次，这条不再妥协 |
| source | **受伤后确立** | 在一段消耗性关系结束后才明确写下来的 |
| status | **生效中** | 正在执行，但还在观察自己能不能守稳 |

**评估属性**：

| 属性 | 取值 | 含义 |
|---|---|---|
| cost | **高代价** | 可能会失去某些关系，也可能要在孤独和边界之间反复挣扎 |

> 两条原则都是"生效中"，但一条是防御性的高代价底线，一条是建设性的低代价标准——这就是 **5 维互不替代**加上**评估属性穿透 5 维**的体现。

#### 4. 第 1 维：6 个原则领域

##### 为什么是 6 个

按**一个成年人做选择时最频繁被调用的 6 个决策面**来切。

| 领域 | 它保护什么 | 典型原则示例 | 看完整性的关键问题 |
|---|---|---|---|
| **关系** | 关系中的对等、边界、真诚 | "不接受单向付出""不替别人做决定""不在背后说重要的话" | "我在关系里守得住自己吗？" |
| **工作** | 精力、尊严、节奏、方向 | "不把工作带进卧室""不为钱做违背判断的事" | "工作在消耗我还是在建设我？" |
| **金钱** | 安全感、自由度、消费观 | "每月存固定比例""不为面子花钱""不借钱投资" | "钱在帮我建立生活还是在制造焦虑？" |
| **健康** | 身体、睡眠、饮食、运动 | "每周至少运动 3 次""12 点前睡觉""不吃情绪化食物" | "身体被放在什么优先级？" |
| **时间** | 节奏、专注、休息权 | "不把周末全排满""不在深夜做决定""每天有独处时间" | "我的时间在被谁支配？" |
| **诚信** | 一致感、自尊、可信度 | "说到做到，做不到就说""不为了被喜欢而撒谎" | "我对自己诚实吗？" |

##### 领域的交叉

一条原则可能同时涉及多个领域。"不把工作带进卧室"既是工作领域（保护下班后的自己），也是健康领域（保护睡眠）。归属规则：**选它最主要保护的领域**。"不把工作带进卧室" → 主要领域是**工作**（它保护的是工作和生活的边界）。

#### 5. 第 2 维：type（原则类型）

`domain` 回答"保护哪个领域"，`type` 回答"以什么方式保护"。

##### 3 种类型

| 类型 | 本质 | 表达方式 | 被突破时的感受 |
|---|---|---|---|
| **底线** | 不可跨过的红线 | "我绝不……""我不能接受……" | "我不再是我了" |
| **边界** | 不许别人/自己进入的区域 | "你不要……""我不会允许……" | "我被越界了" |
| **标准** | 做某类事时的优先做法 | "做 X 时我优先……""我倾向于……" | "我对自己有点失望" |

##### type 不能被 strength 替代

- `type` 回答："这条原则是什么性质？"（红线 / 围墙 / 路标）
- `strength` 回答："它有多稳固？"（不可退让 / 强烈偏好 / 参考指引）

底线的 strength 通常是"不可退让"——但反过来，"不可退让"的也可能是标准（比如有人把"每天跑步"守得像底线一样）。type 和 strength 的相关性很高但不是一一对应，分开能看见反常组合。

#### 6. 第 3 维：strength（强度）

##### 3 级强度

| 强度 | 含义 | 被违反时的动作 |
|---|---|---|
| **不可退让** | 突破这一条等于违背核心自我 | 立即停止 / 退出当前情境 / 严肃修复 |
| **强烈偏好** | 大多数情况遵守，少数情况破例 | 破例后需要复盘 |
| **参考指引** | 大致方向，不强求每次都做到 | 偏离时心里有数，但不苛责自己 |

##### strength 的动态性

原则的 strength 不是固定的。"不接受长期单向付出"可能从"强烈偏好"升级为"不可退让"——当你在旧关系里受了足够的亏之后。反之，一条"不可退让"的原则也可能因为生活阶段变化而降级。

#### 7. 原则的评估属性：cost（代价）

> **重要前提**：cost **不是第 6 个分类维度**，它是穿透所有 5 维的**原则内禀属性**。
> 5 维用来**归类**（按领域聚合、按类型看结构、按强度看优先级），cost 用来**评估单条原则**（守住它要付出什么）。

##### 5 维 vs cost 的区分

| 5 个分类标签 | cost（评估属性） |
|---|---|
| **组织决策体系**：让你能问"关系领域我有哪些原则？" | **判断单条原则**：让你能问"守住这条我真正在付出什么？" |
| 出现在过滤器、分组、统计里 | 跟着每条原则走，在原则详情/决策校准时显示 |
| 用来回答"我有什么标准、在哪、从哪来" | 用来回答"值得吗、我能守多久、我准备好付代价了吗" |

##### 4 级取值

| 等级 | 含义 | 典型原则 | 决策含义 |
|---|---|---|---|
| **高代价** | 守住它可能失去重要关系、机会或资源 | "不接受长期单向付出的关系" | 需要提前想清楚：你准备好失去吗 |
| **中等代价** | 需要持续消耗精力、时间或心力去维护 | "每月存固定比例""每周运动 3 次" | 需要建立配套习惯来降低摩擦 |
| **低代价** | 只需要改变一个操作，几乎没有实质损失 | "不在深夜做决定""不在背后说重要的话" | 低成本高收益，优先固化 |
| **零代价** | 几乎不产生任何摩擦 | "说到做到""不撒谎" | 这种原则通常已经内化，不需要额外付出 |

##### 它能回答的问题

| 问题 | cost 怎么帮 |
|---|---|
| 我准备好守这条底线了吗？ | 先看 cost——高代价的底线需要提前做心理建设 |
| 为什么我总是守不住这条原则？ | 可能 cost 被低估了，实际代价比想象的高 |
| 哪条原则最值得现在就固化？ | 优先 cost=低代价 且 strength 还没到"不可退让"的 |
| 我是不是给自己设了太多高代价原则？ | 高代价占比过高 → 原则体系不可持续 |

##### UI 呈现建议

- ✅ 原则详情页**始终显示** cost
- ✅ 决策校准页显示相关原则的 cost，作为权衡参考
- ✅ 可做"cost × strength"交叉提示（高代价 + 不可退让 → 重点保护）
- ❌ **不要**把 cost 放进主筛选器（那是 5 维的位置）

#### 8. 边界约定

##### 决策规则

> 先问"**这条原则最想保护什么**"，再问"**它是怎么保护这个对象的**"。
> 如果 domain 判断不确定，看"违反它时最先受损的是哪个生活面"。

##### 原则 vs 观念的区分

| | 原则 | 观念 |
|---|---|---|
| **本质** | 对行动的**标准** | 对世界/人生/价值的**看法** |
| **表达方式** | "我以后不再……""我要守住……""做 X 时我优先……" | "世界是……""人生应该……""A 比 B 重要" |
| **作用** | 指导行动、守住边界 | 解释世界、形成判断 |
| **例** | "我不再为不值得的人过度付出" | "人都是自私的" |

观念是原则的上游：很多原则是从观念推导出来的（"人都是自私的" → "所以我要保护自己"）。但也有原则是从直接经历里蹦出来的，中间没有明确的观念——"那次加班到凌晨进医院了" → "不把工作带进卧室"。更多区分见 [观念模块](../beliefs/design.md) §8。

##### 常见归属疑问

| 原则 | 归属 | 理由 |
|---|---|---|
| "不接受单向付出" | 关系 | 保护的是关系中的对等 |
| "不把工作带进卧室" | 工作 | 保护的是工作与生活的边界 |
| "每月存 30%" | 金钱 | 保护的是财务安全感 |
| "12 点前睡觉" | 健康 | 保护的是睡眠 |
| "不在深夜做决定" | 时间 | 保护的是决策质量 |
| "说到做到" | 诚信 | 保护的是可信度和自我一致感 |
| "不借钱给朋友" | 金钱 或 关系 | 核心动机决定：怕丢钱→金钱；怕伤关系→关系 |
| "每周至少独处半天" | 时间 | 保护的是自我恢复的时间 |
| "不在生气时发消息" | 关系 | 保护的是关系不被一时情绪破坏 |
| "不为面子花钱" | 金钱 | 保护的是消费自主权 |

#### 9. 典型问题 → 怎么用回答

| 你的真实困惑 | 用哪几个维度 / 属性 |
|---|---|
| "我到底在守什么？" | 按 domain 分组，看每个领域的 type 分布 |
| "我的底线够不够清楚？" | 过滤 type=底线，按 domain 看——哪些领域没有明确的底线 |
| "哪条原则快守不住了？" | 看 cost=高代价 + strength=不可退让，这些是最脆弱的 |
| "我在保护什么旧伤口？" | 过滤 source=受伤后确立，检查这些原则是否还适用 |
| "这个决定该怎么做？" | 按 domain 调出对应领域的所有 status=生效中 原则，逐条对照 |
| "我的原则体系可持续吗？" | 看 cost 分布——高代价占比过高说明不可持续 |
| "我被越界了吗？" | 过滤 type=底线 + type=边界，对照当前情境 |
| "最近有原则在变吗？" | 过滤 status=正在测试 / 已修订 |
| "哪些原则是别人给我的？" | 过滤 source=家庭继承，看自己是否真的认同 |

#### 10. 实施步骤（给开发者）

1. 把 6 个领域作为 `PrincipleDomain` 枚举写出来
2. 把 3 种类型作为 `PrincipleType` 枚举写出来
3. 把 3 级强度作为 `PrincipleStrength` 枚举写出来
4. 把 4 种来源作为 `PrincipleSource` 枚举写出来
5. 把 4 种状态作为 `PrincipleStatus` 枚举写出来
6. 把 4 级代价作为 `PrincipleCost` 枚举写出来
7. 在 `Principle` 类型上：
   - `domain: PrincipleDomain`（必填）
   - `type: PrincipleType`（必填）
   - `strength: PrincipleStrength`（必填）
   - `source: PrincipleSource`（必填）
   - `status: PrincipleStatus`（必填）
   - `cost: PrincipleCost`（必填）
8. 原则之间支持关联：支撑（一条原则支撑另一条）/ 冲突（两条原则指向不同方向）
9. 每条原则保留修订记录（内容、strength、status 的变化）
10. UI 层注意：**5 维**进入过滤器 / 分组 / 总览统计；**cost** 进详情页和决策校准页
